## 1. 類似プロダクト分析

### 1.1 Canva
**URL**: canva.com
**特徴**:
- テンプレートはすべて「視覚ヒエラルキー」が明確。見出し > サブ見出し > 本文の順に視線が誘導される
- デフォルトの配色は常に2〜3色に絞られており、補色・類似色の組み合わせを多用
- 余白を大胆に使い、要素密度を意図的に低く保つ（50%以上がネガティブスペース）
- タイポグラフィに強い個性を持たせ、フォントサイズの差を「最低2倍以上」に設定することが多い
- ロゴや写真のクリッピングマスクを多用し、幾何学的なレイアウトを演出

**参考になる点**: 余白の使い方、フォントサイズヒエラルキーの大胆さ、3色以内の配色ルール

---

### 1.2 Adobe Express（旧Adobe Spark）
**URL**: adobe.com/express
**特徴**:
- プロフェッショナルなタイポグラフィ設定（文字間隔・行間が精密に調整済み）
- フォントのウェイト（太さ）の組み合わせを使い、色を変えなくても強弱をつける
- 背景画像使用時は必ずオーバーレイを掛けてテキスト可読性を保証
- グリッドシステムを厳格に適用（8pxグリッド）
- 装飾要素はシンプル。線、ブロック、少量の幾何学図形のみ

**参考になる点**: 8pxグリッドの徹底、フォントウェイトによる強弱、装飾のシンプルさ

---

### 1.3 Crello / VistaCreate
**URL**: vista.com/create
**特徴**:
- 「ブランドキット」機能でカラーとフォントを一元管理。バナー間の一貫性が高い
- アニメーション対応のため静止画でも「動き」を感じさせるレイアウト（斜め要素、円弧）
- 写真とテキストの「融合」（テキストが写真を横断するデザイン）が多い
- ハーフトーンパターン、グラデーションメッシュなど質感の豊かな背景
- CTAボタンが必ず視覚的に目立つ位置に配置

**参考になる点**: ブランドカラーの一貫性、写真とテキストの融合手法

---

### 1.4 Fotor
**URL**: fotor.com
**特徴**:
- 写真加工ツール起源のため、背景画像の見せ方が洗練されている
- ビネット（周辺を暗くする）、ブラー、カラーグレーディングを活用した背景処理
- テキストエリアに「グラスモーフィズム」（半透明ガラス風）ブロックを多用
- フォントは最大2種類まで（見出し用セリフ体 + 本文用サンセリフ体）

**参考になる点**: 背景画像処理の手法、グラスモーフィズムによる可読性確保

---

### 1.5 Snappa
**URL**: snappa.com
**特徴**:
- SNSバナー専門ツール。各プラットフォームの仕様を熟知したプリセットが豊富
- テキストは「短く、大きく」が基本。1バナー最大3行という暗黙のルール
- フォントサイズは思い切って大きく（バナー高さの15〜25%が見出しフォントサイズの目安）
- 背景画像を全面使用し、グラデーションオーバーレイで可読性を確保

**参考になる点**: テキスト量の絞り込み、大胆なフォントサイズ設定

### 1.6 BannerSnack / Creatopy
**URL**: creatopy.com
**特徴**:
- 広告バナー専門ツール。CTAの配置がもっとも科学的
- 「3秒ルール」：バナーは3秒で伝わるメッセージに絞る
- ブランドカラーを主役にした配色設計（ブランドカラー70%、ホワイト/ブラック20%、アクセント10%）
- アニメーションバナーのデザインをもとにしているため、要素の「重要度」が明確
- ロゴは右下（または右上）固定が多い（ユーザーが視線を戻す位置）

**参考になる点**: CTAの科学的配置、3秒ルール、ロゴ位置のベストプラクティス

---

### 1.7 Piktochart
**URL**: piktochart.com
**特徴**:
- インフォグラフィック専門起源のため、データの視覚化が洗練されている
- アイコン・イラストを多用したフラットデザイン
- 色のコントラストが強め（黒に近い背景 + 白テキスト or 明るい背景 + 濃いテキスト）
- グリッドベースのレイアウト。要素が水平・垂直に整列している

**参考になる点**: グリッド整列の徹底、コントラストの強さ

---

### 1.8 Visme
**URL**: visme.co
**特徴**:
- プレゼンテーション・バナー兼用ツール。ストーリーテリング重視
- タイポグラフィの階層が3段階（大見出し: 60-80px → 小見出し: 24-32px → 本文: 14-18px）
- 余白を「デザイン要素」として積極活用。要素間の距離感が均一
- 幾何学的な装飾（三角、円、線）をアクセントに使用。ただし主役にはしない
- テキストブロックに一定のパディング（最低16px以上）を設けてテキストが窮屈に見えないよう配慮

**参考になる点**: 3段階タイポグラフィ、余白の均一性、幾何学装飾の脇役扱い

### 1.9 Placeit
**URL**: placeit.net
**特徴**:
- 実際の物体（端末、雑誌等）にはめ込む「モックアップ」特化だが、バナーテンプレートも多数
- 写真とテキストのコントラスト確保のため、写真の一部分にグラデーション矩形を重ねる手法
- ブランドロゴを小さめ（バナー幅の10〜15%）に配置し、テキストを主役にする傾向
- カラーは「ニュートラル背景（白・黒・グレー）+ 1色のブランドカラー」が主流

**参考になる点**: グラデーション矩形でのコントラスト確保、ロゴサイズの適正化

### 1.10 Designbold
**URL**: designbold.com
**特徴**:
- Canvaに近いUXだが、テンプレートは日本語フォントへの対応が充実
- 和文フォントの行間は欧文よりも広め（1.7〜2.0em）が標準
- 縦書きテキストレイアウトのサポート
- アジア圏ビジネス向けバナーのデザインパターンが豊富：赤・金・黒の高級感ある配色

**参考になる点**: 日本語フォントの行間設定、アジアンビジネス向け配色

### ロールモデル選定: **Creatopy (BannerSnack)**

**選定理由**:
1. toBビジネスバナーに特化した設計思想（BannerCreatorのターゲットと一致）
2. 「3秒ルール」という明確な設計原則があり、メッセージの優先度付けが科学的
3. ブランドカラー70% / ホワイト20% / アクセント10%という配色比率が明確で実装しやすい
4. CTAの配置ルールが確立されており、コンバージョンに直結するデザイン
5. コーポレートバナーのプロフェッショナルな見え方を実現している

**SHARKNADOへの取り入れ方**:
- 配色比率（70:20:10）をカラーシステムに明示的に組み込む
- 「3秒ルール」をUIの制約として設定（テキスト文字数ガイドラインの策定）
- ロゴは「補助的存在」として適切なサイズに制約（バナー幅の8〜15%）
- CTAは「右下」か「下部中央」に配置するレイアウトパターンを追加

## 2. プロフェッショナルバナーの共通原則

### 2.1 レイアウト

#### Z型視線誘導の正しい使い方
```
正しいZ型:
① ロゴ（左上）────────────→ ② サブ情報（右上）
      ↘
③ 主要メッセージ（中央・左寄り）
                                      ↘
④ CTA / ブランド（右下）
```

**現状の問題**: BannerCreatorのZ型実装は「4つのゾーンに機械的に分割」しているが、実際のZ型は「視線の流れのパス」であり、各ゾーンを埋めることが目的ではない。

**修正方針**:
- ロゴは左上または右上（視線の起点）に固定
- メインメッセージは中央やや左または中央（視線の中間点）
- 装飾・背景は「Z字の移動を助ける」役割（視線を誘導する矢印的な存在）

#### 余白（ホワイトスペース）のルール
- **要素間距離**: 最小でもフォントサイズの0.75倍以上
- **セーフエリア**: 全辺に対してバナー短辺の8〜12%
- **テキストブロック内パディング**: フォントサイズの0.5倍以上（上下左右均等）
- **ロゴクリアスペース**: ロゴ高さの50%以上を四方に確保

#### 要素間の距離感（8pxグリッド）
| 関係 | 推奨距離 |
|------|----------|
| 密接（同グループ） | 8px |
| 近接（関連要素） | 16px |
| 分離（異なるグループ） | 32px |
| セクション区切り | 48px以上 |

**現状の問題**: layout.jsonでは絶対px値ではなくバナー幅に対する比率のみで管理されており、8pxグリッドの概念が欠けている。

---

### 2.2 タイポグラフィ

#### フォントサイズの比率
プロフェッショナルなバナーの標準比率:

| 要素 | 比率（見出しを1とした場合） | BannerCreator推奨値 |
|------|--------------------------|---------------------|
| 大見出し | 1.0（基準） | バナー幅の5〜7% |
| 小見出し / キャッチ | 0.618（黄金比） | バナー幅の3〜4% |
| 本文 / 補足 | 0.382（黄金比²） | バナー幅の2〜2.5% |
| 極小（法律表記等） | 0.236 | バナー幅の1.2〜1.5% |

**現状の問題**:
- `layoutCalculator.ts` の `titleFontSizeRatio.max = 0.06` は妥当
- しかし `subtitleFontSizeRatio.max = 0.035` はタイトルの58%（黄金比の0.618に近いが若干小さい）
- **キャプション・補足テキストが定義されていない**（2要素のみのヒエラルキー）

#### 文字間隔（Letter Spacing）
| テキスト種別 | 推奨値 |
|------------|--------|
| 日本語見出し（太字） | 0.02〜0.05em |
| 日本語本文 | 0.01〜0.02em |
| 英語アクセントテキスト（装飾用） | 0.15〜0.3em（大きく広げる） |
| 英語見出し | 0.05〜0.1em |

**現状の問題**: `layoutCalculator.ts` の `letterSpacing: titleFontSize * 0.05` は日本語見出しとして適切。ただしenerkeyスタイルでは `letterSpacing: 0` となっており統一されていない。

#### 行間（Line Height）
| テキスト種別 | 推奨値 |
|------------|--------|
| 日本語大見出し | 1.3〜1.5em |
| 日本語中見出し | 1.5〜1.7em |
| 日本語本文 | 1.7〜2.0em |
| 英語見出し | 1.1〜1.3em |

**現状の問題**: `layoutCalculator.ts` の `titleLineHeight = titleFontSize * 1.3` はやや行間が狭い。日本語では1.4〜1.5が自然。

#### フォントウェイトの使い分け
- 大見出し: **700（Bold）** または **900（Black）**
- 小見出し / キャッチ: **500（Medium）** または **600（SemiBold）**
- 本文: **400（Regular）**
- アクセント強調: **700（Bold）+ アクセントカラー**（色とウェイトを同時に変える）

**現状の問題**: クラシックレイアウトはタイトルのみ700。Enterkeyレイアウトはセグメントごとに400/700を切り替えられるが、500/600ウェイトの活用が不足している。

### 2.3 配色

#### 3色ルールの具体的な使い方

**60-30-10ルール**（業界標準）:
```
60%: ドミナントカラー（背景、オーバーレイ）
     → 最も落ち着いた色。ブランドカラーのダーク系統
30%: セカンダリカラー（テキスト、ロゴ背景）
     → 白または明るいニュートラル
10%: アクセントカラー（CTA、強調帯、装飾）
     → 彩度が高く目立つ色
```

**HSL明度・彩度の揃え方**:
- メインカラーとアクセントカラーの**明度差を20〜40%**に設定
- 彩度は同程度（±15%以内）に揃えると統一感が生まれる
- アクセントカラーはメインカラーと**補色（180°）またはトライアド（120°）**関係にある色を選ぶ

**具体例**（Enterkey風プリセット）:
```
ブランドカラー: #1A1A2E（Hue: 240°, S: 29%, L: 14%）
アクセント:    #E94560（Hue: 352°, S: 80%, L: 58%）

問題: 明度差 44%（大きすぎる）、彩度差 51%（大きすぎる）
→ アクセントが浮きすぎて安っぽく見える原因
```

**改善案**:
```
ブランドカラー: #1A1A2E（H: 240°, S: 29%, L: 14%）
アクセント:    #C0392B（H: 4°, S: 63%, L: 46%）
→ 彩度差34%、明度差32%に収まり、統一感が増す
```

#### 背景とテキストのコントラスト
- WCAG AA基準: 4.5:1以上（通常テキスト）
- WCAG AA基準: 3:1以上（大きなテキスト18px以上または太字14px以上）
- **実用的な目安**: オーバーレイ不透明度0.5以上で白テキストはほぼ保証される

**現状の問題**:
- `overlayDefault: 0.5` は最低ラインとして妥当
- しかし**背景画像の輝度によってオーバーレイ不透明度が自動調整される仕組みが無い**
- 明るい背景写真 + 0.5オーバーレイでは白テキストが読めない場合がある

#### オーバーレイの適切な濃度

| 背景写真の特性 | 推奨オーバーレイ不透明度 |
|-------------|----------------------|
| 暗い写真（夜景、ダークスタジオ） | 0.3〜0.4 |
| 中程度の写真（標準的な屋外） | 0.5〜0.6 |
| 明るい写真（白背景、昼間屋外） | 0.65〜0.8 |
| パターン画像（複雑な模様） | 0.7〜0.85 |

### 2.4 装飾

#### 幾何学模様の効果的な使い方
プロフェッショナルなバナーでは、装飾は以下の「役割」を果たす:
1. **視線誘導**: 斜め線、矢印形の要素で視線をCTAへ向かわせる
2. **ブランド強化**: 企業のシンボルや幾何学アイデンティティを反映
3. **奥行き感**: 複数レイヤーの装飾で立体感を演出

#### 装飾の量（少ない方がプロフェッショナル）
**装飾量の黄金律**: バナー面積の**5〜15%以内**に装飾要素を収める

| 装飾量 | 印象 |
|-------|------|
| 0〜5% | ミニマル・洗練・高級感 |
| 5〜15% | バランス・プロフェッショナル（推奨） |
| 15〜30% | 賑やか・カジュアル |
| 30%以上 | うるさい・安っぽい |

**現状の問題**:
- ストライプ枠（`stripe-border`）は面積は少ないが、ダッシュ状の枠が「安いチラシ感」を与えている
- ストライプ枠のデザインがプロフェッショナルに見えない理由: `stroke-dasharray="10,5"` は均等すぎる
- **改善**: 実線の細い枠（2px以下）または角部分のみのL字装飾の方がハイエンドに見える

#### 装飾の配置ルール
- 装飾は**コーナー**か**エッジ**に配置（中央に装飾を置くと主役コンテンツと競合する）
- 装飾の色はアクセントカラーを使うが、**不透明度を0.3〜0.6に下げる**ことで主役を邪魔しない
- 装飾同士が対角線上に配置されると「バランスが取れている」印象を与える

**現状の問題**:
- Enterkeyスタイルのコーナー三角装飾（`stripe-triangle-corner`）は良い配置だが、**ストライプパターンが細かすぎる**（`stripeWidth: 4, stripeGap: 6`）
- サイズが大きい（`cornerTriangleSize: 0.12`）ため、三角形が「装飾」でなく「主役」になってしまっている

### 2.5 画像処理

#### 背景画像の見せ方
- **クロップの方向**: `xMidYMid slice` はCanvasの標準設定として正しい
- **焦点調整**: ブランド写真では人物の顔・商品が切れないよう焦点位置を設定すべき（現状: 中央固定のみ）
- **色調統一**: 背景写真にブランドカラーのカラーオーバーレイを掛けると統一感が生まれる

**プロフェッショナルな写真処理手法**（各ツールの共通点）:
1. グラデーションオーバーレイ（写真上に単色ではなくグラデーション）
2. テキストがある側を暗く、写真が見える側を明るく（Enterkeyスタイルで実装済み）
3. ビネット効果（周辺を暗くする）を軽く掛けて中央を際立たせる

#### ロゴの適切なサイズと配置
| バナーサイズ | ロゴ推奨最大幅 |
|------------|--------------|
| 1200×628（OGP） | 200〜240px（幅の17〜20%） |
| 1080×1080（SNS正方形） | 180〜220px（幅の17〜20%） |
| 1200×400（横長バナー） | 160〜180px（幅の13〜15%） |
| 728×90（広告バナー） | 80〜100px（幅の11〜14%） |

**現状の問題**:
- `logoMaxWidth = safeArea.width * 0.45`（セーフエリアの45%）は大きすぎる
- 1200pxバナーのセーフエリア幅は960px → ロゴ最大幅が432px（バナー幅の36%）になる
- これはロゴがメインコンテンツを圧迫する原因になる
- 推奨: `logoMaxWidth = safeArea.width * 0.20〜0.25`（バナー幅の18〜22%程度）

#### 写真クリップの効果的な使い方
- 斜めクリップ（平行四辺形）は「動き」「スピード感」を演出する
- 角度は**8〜15度**が最適（小さすぎると効果が弱く、大きすぎると不安定に見える）
- クリップエリアは全体の**55〜65%**が適切（現状の0.65は上限値）

## 3. 現在のBannerCreatorの問題点と改善提案

### 3.1 レイアウト問題

#### 問題1: ロゴが大きすぎる
```
現状コード (layoutCalculator.ts: L56):
const logoMaxWidth = safeArea.width * 0.45;
const logoWidth = Math.max(width * layoutParams.logoMinSizeRatio, logoMaxWidth);
```

**問題**: logoMinSizeRatioが0.08（96px at 1200px）、logoMaxWidthが0.45のセーフエリア幅（432px）と極端な差がある。`Math.max` でlogoMaxWidthを選ぶため、ロゴは常に432pxになる（バナー幅の36%）。

**改善案**:
```json
"logoMaxSizeRatio": 0.22,  // バナー幅の22%（264px at 1200px）
"logoMinSizeRatio": 0.10,  // バナー幅の10%（120px at 1200px）
```

#### 問題2: タイトルの行間が狭い
```
現状コード (layoutCalculator.ts: L97):
const titleLineHeight = titleFontSize * 1.3;
```

**問題**: 1.3は英語テキスト向けの行間。日本語の可読性には1.4〜1.6が推奨される。

**改善案**:
```typescript
const titleLineHeight = titleFontSize * 1.5; // 日本語向けに1.5に変更
```

#### 問題3: タイトル配置のY座標計算が複雑で意図不明
```
現状コード (layoutCalculator.ts: L99):
const titleY = safeArea.y + minorH + (majorH - titleLineHeight * 2) * 0.6;
```

**問題**: `(majorH - titleLineHeight * 2) * 0.6` の `0.6` という係数に設計意図がなく、バナーサイズが変わると位置がバラバラになる。

**改善案**: Z型パターンのゾーンを使って配置位置を決定する（既にcalculateZPatternZonesを実装しているのに使われていない）。

#### 問題4: 装飾のストライプ枠がチープに見える
```
現状コード (svgGenerator.ts: L157):
stroke-dasharray="10,5"
```

**問題**: ダッシュ状の枠線はチラシ・フライヤーの安価なイメージを連想させる。

**改善案**:
- 実線の細い枠（`stroke-width="2"` 、`stroke-dasharray`なし）に変更
- または「L字型のコーナー装飾」（四隅のみの線）に変更
- エンタキースタイルと同様のコーナー三角デザインに統一

### 3.2 タイポグラフィ問題

#### 問題5: フォントサイズのヒエラルキーが2段階しかない
現状: タイトル（大）+ ハイライトキーワード（同じサイズ）の2要素のみ。

**改善案**: 3段階ヒエラルキーを導入:
- 見出し（大）: バナー幅の5〜7%
- サブ見出し（中）: バナー幅の3〜4%（見出しの60%サイズ）
- 補足テキスト（小）: バナー幅の1.8〜2.5%（見出しの35%サイズ）

#### 問題6: Enterkeyスタイルでletter-spacingが0
```
現状コード (layoutCalculator.ts: L362):
letterSpacing: 0,
```

日本語テキストでletter-spacingが0は文字が詰まりすぎてプロフェッショナルに見えない。

**改善案**: `letterSpacing: titleFontSize * 0.03` に設定

### 3.3 配色問題

#### 問題7: 明度差が大きすぎるデフォルトプリセット
```
プリセット例: brandColor: '#1A1A2E', accentColor: '#E94560'
明度差: 14% vs 58% → 差44%（推奨は20〜40%）
彩度差: 29% vs 80% → 差51%（推奨は±15%以内）
```

これが「安っぽく見える」最大の原因。

**改善案**:
- 各プリセットのアクセントカラーをブランドカラーと明度・彩度が揃った色に変更
- colorGenerator.ts の `generateDerivedColors` でアクセントカラーの明度・彩度を自動補正する機能を追加

#### 問題8: オーバーレイが一律固定（0.5）
背景写真の輝度に関わらず `overlayDefault: 0.5` 固定。

**改善案**: 背景画像の平均輝度を計算し、オーバーレイ不透明度を自動調整する。
```
推奨値:
暗い背景（平均輝度 < 0.3）: overlayOpacity = 0.3
中程度（0.3 ≤ 輝度 < 0.6）: overlayOpacity = 0.5
明るい背景（輝度 ≥ 0.6）:   overlayOpacity = 0.7
```

### 3.4 装飾問題

#### 問題9: コーナー三角が大きすぎる
```
現状: cornerTriangleSize: 0.12（バナー幅の12%）
1200pxバナーでは144px角の三角形
```

**改善案**: `cornerTriangleSize: 0.08`（バナー幅の8%）に縮小。96px角で「装飾」として機能する適切なサイズ。

#### 問題10: 背景なし時のグラデーションが複雑すぎる
```
現状: 4ストップのlinearGradient + radialGradient（グロー）+ ドットパターン
```

複数のレイヤーが重なってノイジーになる場合がある。

**改善案**: 2ストップのシンプルなグラデーション（ライト → ダーク）に統一し、グロー効果のみ追加。

## 5. ロールモデル（Creatopy）から学ぶ改善優先度

### 優先度高（即時改善）
1. ロゴサイズの適正化（最大幅をバナー幅の22%に制限）
2. 配色の明度・彩度を揃える（プリセットのアクセントカラー見直し）
3. ダッシュ枠装飾を実線または非表示に変更
4. 日本語の行間を1.3 → 1.5に修正

### 優先度中（短期改善）
5. 3段階タイポグラフィヒエラルキーの導入
6. 背景輝度に基づくオーバーレイ自動調整
7. コーナー三角のサイズ縮小（0.12 → 0.08）
8. Enterkeyレイアウトのletter-spacing追加

### 優先度低（中長期改善）
9. 背景なし時のグラデーションをシンプル化
10. 焦点調整機能（背景写真のfocal point設定）
11. CTAエリアの追加（現状のレイアウトにはCTAの概念がない）
