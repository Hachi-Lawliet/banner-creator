# PJメモ

# Webアプリ「BannerCreator」：概要

---

- ゴール
    - **toB向けサービスのWebサイト用バナーを、デザインルールに則って効率的に量産できるツールの作成**
- 施策概要
    - Webアプリの作成
        - デスクトップがメイン（社内ツール）
        - Next.js / React.js / TailwindCSS
        - PNG/SVGダウンロードのみ（Figma連携は後フェーズ）

# 価値定義

---

### ① ステークホルダー

| ロール名 | 説明 | 権限 |
| --- | --- | --- |
| バナー作成者 | 社内でtoB向けサービスのバナーを制作する担当者 | ・テンプレート選択 ・素材アップロード ・バナー生成 ・Figmaへの書き出し ・バナーサイズ指定 |

### ② イシュー

- バナーを作るたびにデザインルール（配色・配置・バランス）を手動で守るのが面倒
- デザイナーでなくてもルールに沿ったバナーを作りたいが、ツールがない
- バナーサイズが変わるたびにレイアウトを手動で調整し直すのが非効率
- Figmaでの微調整を前提としつつ、ベースのレイアウトは自動で整えたい

### ③ ソリューション

- デザインルール（配色・重心・視線誘導・黄金比）を内蔵したバナー生成ツール
- ユーザーはテンプレート選択 → 素材入力 → サイズ指定するだけで、ルールに準拠したバナーが自動生成される
- 生成されたバナーはPNG/SVG形式でダウンロード可能（Figma連携は後フェーズ）

# アプリについて

---

## アプリ概要

- Next.js / React.js / TailwindCSS で構築
- デスクトップブラウザでの利用を想定
- PNG/SVGダウンロード機能（Figma連携は後フェーズ）

## UIデザイン概要

- クリーンでシンプルなツール系UIデザイン
- プレビュー領域を広く取り、リアルタイムでバナーの仕上がりを確認できる
- 左サイドに入力パネル、右側にプレビューのレイアウト
- Tailwind CSSによるコンポーネント設計

## デザインルール（バナー生成エンジンが制御）

### 配色ルール

- **ブランドカラー**: ユーザーが1色を指定
- **派生カラー**: ブランドカラーから明度・彩度を変えた3色を自動生成
    - ライト（明度+20%）: 背景・オーバーレイ用
    - ミディアム（原色）: メインテキスト・帯用
    - ダーク（明度-20%）: 強調・ボーダー用
- **アクセントカラー**: ユーザーが任意で指定可能
    - バナー全体面積の10〜15%以内に制限
    - キーワード強調帯に使用
- **オーバーレイ**: 背景画像の上に半透明のブランドカラー（ダーク）を重ねてテキスト視認性を確保
    - オーバーレイ濃度はスライダーUI（0-80%、デフォルト50%）でユーザー調整可能

### 配置・バランスルール

- **黄金比分割**: バナーの主要領域を1:1.618の比率で分割
- **重心制御**: バナーの視覚的重心が中央〜やや上に来るよう自動調整
- **視線誘導**: Z型パターンに沿った要素配置
    - 左上 → 右上 → 左下 → 右下 の順で視線が流れるよう設計
- **セーフエリア**: バナー端から10%のマージンを確保し、テキスト・ロゴの見切れを防止
- **比率ベースレイアウト**: すべての要素サイズ・位置を比率で定義し、任意のバナーサイズに対応

### フォント

- 固定の1書体を使用（具体的な書体はデザインフェーズで決定）
- サイズはバナー幅に比例してスケール

### 見切れ防止

- テキスト: コンテナはコンテンツに合わせてサイズ確保、文字数が多い場合はフォントサイズを自動縮小
- ロゴ: アスペクト比を維持したまま、指定エリア内にフィット
- 人物写真: 顔検出（将来的な拡張）or セーフエリアを広めに取って対応

## テンプレート

### テンプレートB（ロゴ主役型）- 1種類のみ

参考: 「ハイッテ」Tent.バナーデザイン

| 要素 | 配置 | 説明 |
| --- | --- | --- |
| クライアントロゴ | 上部（目立つ位置） | ロゴレギュレーションに違反しないサイズ・余白を確保 |
| タイトル | 中央 | 2行対応、キーワードをアクセントカラー帯で強調 |
| 背景画像 | 全面（サブ的に使用） | オーバーレイを重ねて視認性確保 |
| 装飾 | 適宜 | ストライプパターン枠、英語装飾テキスト（例：「Case studies」） |
| キーワード強調 | タイトル内 | アクセントカラーの帯でキーワードを強調 |

**改善ポイント（参考画像の課題を解消）**:
- 重心が下に偏らないよう、ロゴ〜タイトルの配置バランスを自動調整
- アイキャッチが弱くならないよう、背景画像の露出度を最適化

## チーム会議で採用された技術的決定事項

### Canvas描画レイヤー順
1. 背景色
2. 背景画像
3. オーバーレイ
4. 装飾
5. タイトル
6. ロゴ

### テキスト自動縮小
- 二分探索で最適フォントサイズを特定

### 重心計算式
```
重心Y = Σ(Y中心 × 面積 × 色濃度) / Σ(面積 × 色濃度)
目標: 40-50%
```

### レイアウトパラメータ
- JSON設定ファイルとして外部化

### コントラスト比チェック
- WCAG AA基準: 4.5:1以上

### 装飾要素
- ON/OFF切替可能
