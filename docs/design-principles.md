# バナーデザイン基本原則

本ドキュメントはBannerCreatorのレイアウトエンジンが参照するデザイン原則をまとめたものです。
すべてのバナー生成時にこのルールを適用します。

リサーチ更新日: 2026-03-22（design-research.md に基づく改訂版）
ロールモデル: Creatopy (BannerSnack) - toBビジネスバナー特化ツール

---

## 1. 黄金比（Golden Ratio: 1:1.618）

### 原則
- バナーの主要領域の分割に黄金比を使用する
- 例: 幅1200pxのバナーなら、主要コンテンツ領域を741px : 459pxに分割

### 適用箇所
- **縦分割**: 画像領域とテキスト領域の比率
- **余白**: メイン要素とサブ要素の間隔
- **フォントサイズ**: タイトル : サブタイトル = 1.618 : 1（黄金比）、本文 = サブタイトルの0.618倍

### バナーへの実装
```
メイン領域幅 = バナー幅 × (1.618 / 2.618) ≈ 61.8%
サブ領域幅   = バナー幅 × (1 / 2.618)     ≈ 38.2%
```

## 2. 重心（Visual Weight & Balance）

### 原則
- バナーの視覚的重心は**中央〜やや上（40〜48%）**が安定する
- 重い要素（大きい文字、ロゴ、濃い色）が偏ると不安定に見える
- 上部に重心があると「信頼感」「安定感」、下部だと「不安定」「落ち着かない」印象

### 重心の計算方法
各要素の「視覚的重さ」を以下で算出:
- **サイズ**: 面積が大きいほど重い
- **色の濃さ**: 暗い色・彩度の高い色ほど重い
- **位置**: 端に近いほど重心への影響が大きい（てこの原理）

### 重心計算式（実装用）
```
重心Y = Σ(Y中心 × 面積 × 色濃度) / Σ(面積 × 色濃度)
目標範囲: 40-48%（バナー高さに対する比率）
※ 50%より上にずらすことで視覚的な安定感を向上
```

### 5つの重心ポジション
| ポジション | 印象 | 使用シーン |
| --- | --- | --- |
| 上 | 軽快・開放的 | 写真主役で下部にテキスト |
| 中央やや上（推奨） | 安定・フォーマル・信頼 | toBビジネスバナーの標準 |
| 中央 | 均等・無個性 | 汎用テンプレート |
| 下 | 安定だがやや重い | 要注意（Bテンプレの課題） |
| 左/右 | 読み始め/結論 | Z型レイアウトの起点・終点 |

### Bテンプレートでの対策
- ロゴ（上部）で重心を引き上げる
- タイトル帯の位置を中央〜やや上に配置
- 背景画像のオーバーレイ濃度で重さを微調整

## 3. 視線誘導（Eye Flow）

### Z型パターン（横長バナー向け）
```
① 左上（ロゴ・ブランド） ─────→ ② 右上（装飾・補足）
         ↘
③ 中央（メインメッセージ・タイトル）
                                          ↘
④ 右下（CTA・問い合わせ先）
```

**重要**: Z型は「4つのゾーンに要素を埋める」のではなく、「視線の移動パスを設計する」こと。

### Z型要素の優先度
- **①左上（最高優先度）**: ロゴ。バナーを見た瞬間に目がいく位置
- **③中央（高優先度）**: タイトル + キーワード強調。最も伝えたい情報
- **②右上（中優先度）**: 装飾または補足情報。視線の流れを助ける
- **④右下（CTA位置）**: 行動喚起テキストまたはブランド情報

### F型パターン（縦長バナー向け）
```
① ━━━━━━━━━━━━（最重要情報を上に）
② ━━━━━━━（次に重要な情報）
③ ┃（左側に視線が収束）
   ┃
   ↓
```

- 重要な情報を上部に集約
- 左側に重要要素を配置

### Bテンプレートへの適用
1. **①左上〜中央上**: クライアントロゴ（最初に目がいく）
2. **③中央**: タイトル + キーワード強調帯
3. **背景全体**: 写真（サブとして認識）
4. **②右上 / ④右下**: 装飾でZ字の動きを補助

## 4. 見切れ防止（Safe Area）

### セーフエリアの定義
```
┌──────────────────────────────┐
│  8〜12% margin (safe area)   │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │   コンテンツ領域      │    │
│  │                      │    │
│  └──────────────────────┘    │
│                              │
└──────────────────────────────┘
```

### ルール
- **テキスト**: セーフエリア内に完全に収まること
- **ロゴ**: セーフエリア内に収め、さらにロゴ独自の余白（クリアスペース）を確保
  - ロゴクリアスペース = ロゴ高さの50%以上（四方）
- **背景画像**: セーフエリア外にはみ出してOK（むしろ端まで使う）
- **装飾要素**: セーフエリアの境界・コーナーに配置可（ストライプ枠など）

### テキスト見切れ防止の実装
1. テキストをレンダリングし、実際の描画サイズを計測
2. コンテナ幅を超える場合:
   - **二分探索で最適フォントサイズを特定**（段階的縮小より高速）
   - それでも収まらない場合は改行
   - 最大行数を超える場合はエリプシス（...）

## 5. 配色の基本（Color Theory）

### 60-30-10ルール（業界標準）
| 比率 | 用途 | BannerCreatorでの適用 |
| --- | --- | --- |
| 60% | ドミナントカラー（最も落ち着いた色） | 背景・オーバーレイ（ブランドカラーのダーク〜ミディアム） |
| 30% | セカンダリカラー（中立的） | テキスト・ロゴエリア（白または明るいニュートラル） |
| 10% | アクセントカラー（最も目立つ） | CTA・強調帯・装飾 |

### 明度・彩度の揃え方（HSLベース）

**3色の関係性ルール**:
```
ブランドカラー（H, S, L）
  ↓
  ライト:    H, S-10%, L+20%   → 背景・オーバーレイ用（明るい派生）
  ミディアム: H, S,    L        → メインカラー（原色）
  ダーク:    H, S+5%, L-20%    → 強調・ボーダー用（暗い派生）

アクセントカラー:
  → ブランドカラーと補色（H+180°）またはトライアド（H+120°）関係
  → 明度差: ブランドカラーとの差は 20〜40%以内（差が大きすぎると安っぽく見える）
  → 彩度差: ブランドカラーとの差は ±20%以内に収める
```

**悪い例（現状プリセットの問題）**:
```
ブランドカラー: #1A1A2E（H: 240°, S: 29%, L: 14%）
アクセント:    #E94560（H: 352°, S: 80%, L: 58%）
明度差: 44%（大きすぎる → アクセントが浮いて安っぽく見える）
彩度差: 51%（大きすぎる → チープな印象）
```

**良い例（改善後）**:
```
ブランドカラー: #1A1A2E（H: 240°, S: 29%, L: 14%）
アクセント:    #C0392B（H: 4°, S: 63%, L: 46%）
明度差: 32%、彩度差: 34%（許容範囲内 → 統一感のある配色）
```

### コントラスト比
- テキストと背景のコントラスト比は**4.5:1以上**を確保（WCAG AA基準）
- 大きなテキスト（18px以上）: 3:1以上（WCAG AA）
- オーバーレイの不透明度で自動調整

### オーバーレイ推奨濃度（背景輝度別）
| 背景写真の特性 | 推奨オーバーレイ不透明度 |
|-------------|----------------------|
| 暗い写真（夜景・ダークスタジオ） | 0.3〜0.4 |
| 中程度の写真（標準的な屋外） | 0.5〜0.6 |
| 明るい写真（白背景・昼間屋外） | 0.65〜0.8 |
| パターン画像（複雑な模様） | 0.7〜0.85 |

## 6. タイポグラフィ

### フォント
- **書体**: Noto Sans JP（Google Fonts）
- **ローディング**: `document.fonts.load()` で読み込み完了を待ってからレンダリング

### フォントサイズのスケーリング（3段階ヒエラルキー）
バナー幅に対する比率で定義:
```
大見出し（タイトル）:    バナー幅の 5〜7%  （1200pxで 60〜84px）
中見出し（サブタイトル）: バナー幅の 3〜4%  （1200pxで 36〜48px）= 大見出しの 0.618倍
補足テキスト:           バナー幅の 1.8〜2.5%（1200pxで 22〜30px）= 大見出しの 0.35〜0.4倍
```

### 行間（Line Height）
- 日本語大見出し: **1.4〜1.5em**（現状の1.3は狭すぎる → 要修正）
- 日本語中見出し: 1.5〜1.7em
- 日本語補足テキスト: 1.7〜2.0em
- 英語アクセントテキスト: 1.1〜1.3em

### 文字間（Letter Spacing）
| テキスト種別 | 推奨値 |
|------------|--------|
| 日本語大見出し（太字） | fontSize × 0.03〜0.05 |
| 日本語中見出し | fontSize × 0.02〜0.03 |
| 英語装飾テキスト（小さい） | fontSize × 0.15〜0.3（大きく広げる） |
| 英語見出し | fontSize × 0.05〜0.1 |

### フォントウェイトの使い分け
| 用途 | ウェイト |
|------|---------|
| 大見出し | 700（Bold）|
| 強調キーワード | 700（Bold）+ アクセントカラー |
| 中見出し | 500〜600（Medium/SemiBold）|
| 本文 | 400（Regular）|
| 極小テキスト | 300〜400（Light/Regular）|

## 7. ロゴレギュレーション

### サイズルール（改訂版）
| バナーサイズ | ロゴ最大幅 | ロゴ最小幅 |
|------------|-----------|-----------|
| 1200×628 | 264px（バナー幅の22%） | 120px（10%）|
| 1080×1080 | 238px（22%） | 108px（10%）|
| 1200×400 | 216px（18%） | 96px（8%）|
| 728×90 | 87px（12%） | 44px（6%）|

**現状からの変更**: `logoMaxWidth = safeArea.width * 0.45`（432px）→ `バナー幅 * 0.22`（264px）

### 配置ルール
- **最優先**: 左上（Z型視線誘導の起点）
- **代替**: 右上（視線が自然に流れる終点）
- **避けること**: 中央配置（テキストと競合し視線が分散する）
- **クリアスペース**: ロゴ高さの50%以上を四方に確保
- **アスペクト比**: 絶対に変更しない

## 8. 装飾の原則

### 装飾量の基準
| 装飾量（バナー面積に対して） | 印象 |
|--------------------------|------|
| 0〜5% | ミニマル・洗練・高級感 |
| 5〜15% | バランス・プロフェッショナル（推奨） |
| 15〜30% | 賑やか・カジュアル |
| 30%超 | うるさい・安っぽい |

### 装飾のルール
- 装飾はコーナーまたはエッジに配置（中央は禁止）
- 装飾の不透明度: アクセントカラー使用時は0.3〜0.7に抑える
- 対角線上の装飾配置でバランスを取る（例: 右上 + 左下のコーナー装飾）
- **ダッシュ状の枠線は使わない**（安価なチラシ感が出る）
- 代わりに使うもの: 実線（2px以下）、L字コーナー装飾、コーナー三角

### コーナー三角のサイズ基準
- 推奨サイズ: バナー幅の**8%**（1200pxで96px）
- 現状: 12%（144px）→ 縮小が必要
- 不透明度: 0.5〜0.7が適切

## 9. enterkey風レイアウト原則

### 構造
- 左側: 斜めクリップ（平行四辺形）で写真を配置（全幅の55〜65%）
- 右側: ダーク背景上にリッチテキスト
- コーナー: ストライプ三角装飾（右上・左下）

### 斜めクリップの推奨角度
- 最小: 8度（動きが弱い）
- 推奨: 10〜12度（バランスが良い）
- 最大: 15度（スピード感あり）
- 現状設定: 15度（問題なし）

### リッチテキスト
- 行単位でセグメント分割（テキスト + フォントウェイト + 色）
- 通常テキスト（weight: 400）と強調テキスト（weight: 700 + アクセントカラー）の使い分け
- フォントサイズはバナー幅の5〜6%を基準に、テキスト長で自動縮小
- **letter-spacing**: `fontSize * 0.03`（現状の0は要修正）

### グラデーションオーバーレイ
- 写真上に左→右のグラデーション（opacity: 0.7 → 0.3 → 0）
- テキスト側のダーク背景との自然な境界を形成

## 10. テキストの可読性

### テキストシャドウ
- Canvas: `shadowColor`, `shadowBlur`, `shadowOffsetX/Y` を使用
- SVG: `<feDropShadow>` フィルターを使用
- パラメータ: `rgba(0,0,0,0.45)`, blur = fontSize * 0.08, offsetX = 1, offsetY = 2

### 背景なし時のデザイン
- 2色グラデーション（ライト → ダーク）をベースにシンプルに
- 放射状グロー（中央やや左、opacity 15〜20%）でアクセント
- 過剰なレイヤーは避ける（現状の4ストップグラデーション + グロー + ドットパターンは複雑すぎる）

## 12. 実装優先度（リサーチに基づく改善一覧）

### 優先度A（即時修正必須）
1. **ロゴサイズ**: `safeArea.width * 0.45` → `バナー幅 * 0.22`（logoMaxWidth計算の変更）
2. **日本語行間**: `titleFontSize * 1.3` → `titleFontSize * 1.5`
3. **装飾枠**: ダッシュ状（stroke-dasharray）を廃止 → 実線またはコーナーL字に変更
4. **アクセントカラー**: 各プリセットの明度差・彩度差を20〜40%に調整

### 優先度B（短期改善）
5. **letter-spacing**: Enterkeyスタイルで`fontSize * 0.03`を追加
6. **コーナー三角サイズ**: `0.12` → `0.08`（バナー幅比率）
7. **オーバーレイ自動調整**: 背景輝度計算に基づく自動設定
8. **3段階タイポグラフィ**: 大見出し・中見出し・補足の3レベルを型定義に追加

### 優先度C（中長期改善）
9. 背景グラデーションのシンプル化（2ストップ + グロー1層のみ）
10. 焦点点調整機能（背景写真のfocal point指定）
11. CTAエリアのレイアウト対応
